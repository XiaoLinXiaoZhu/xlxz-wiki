存在下面几个问题：
### 1. 没有忽略 代码块

代码块中的内容应该被视为普通文本，不应该被解析为模板或其他特殊语法。

但是就现在而言，如果是：
```
【定义项】：定义内容
```
也会被解析认为这是一个 文件内定义 （内联定义），而不是普通文本。

### 2. 索引 和 非索引

似乎在现在的实现中，两种定义模式：
1. 单文件定义，通过 元数据 的 alias 指定定义的词条，整个文档作为一个定义，并且支持 scope 指定作用域
2. 内联定义，通过 `【定义项】：定义内容` 语法直接在文档中定义词条，但是这种词条只能在当前文档中有效。

这两种定义模式均需要首先生成一个索引 之后才能够正确展示定义内容。

我能够理解 单文件定义模式 需要索引，因为它是跨文档的定义，需要在整个项目中查找。且拥有复杂的作用域和别名支持。

但是内联定义模式 似乎不需要索引，因为它只在当前文档中有效。

我想就是说，我们的内联定义可不可以不需要索引，而是直接通过某种方式从文档中提取定义内容。但是同时也要保证之前已有的功能不变。

这样这样的话，每次更新单个文档是，不需要重新启动服务来生成索引，而是直接能够通过热更新来更新当前文档的内联定义。
