## 模板功能的重新声明

首先重新声明一下，模板并不是为了重复利用，而是为了能够提供更好的匹配和展示效果，比如：
ps: 它也应该兼容 alias 声明，而不只是 内联定义，这里使用内联定义是为了方便

```markdown
玩家A 攻击 玩家B
```

这种情况，如果是按照 词条定义来处理，那么我们需要这样处理以展示

```markdown
【攻击】：某个人攻击了某个人，之后需要进行伤害的结算和各种buff的效果结算

玩家A【攻击】玩家B
```

这里的 某个人 攻击 某个人，非常愚蠢，它带来了额外的阅读负担。

但是如果我能够：
```markdown
【{A} 攻击 {B}】：{A}攻击了{B}，之后需要进行伤害的结算和各种buff的效果结算

【{玩家A} 攻击 {玩家B}】
```

之后我将鼠标悬停上去，它能够显示
```markdown
玩家A 攻击了 玩家B，之后需要进行伤害的结算和各种buff的效果结算
```
那么就非常好。


更进一步了，我希望在这里面能够使用嵌套定义
```markdown
【{A} 对 {B} 施加了 {C}】：{A}对{B}施加了{C}，之后需要根据 {C} 的 具体效果 来进行伤害的结算和各种buff的效果结算
【火球术】：火系法术，造成火焰伤害，并有几率附加灼烧效果

【{玩家A} 对 {玩家B} 施加了 {【火球术】}】
```

之后，我将鼠标悬停上去，它能够显示
```markdown
玩家A 对 玩家B 施加了 【火球术】，之后需要根据 【火球术】 的 具体效果 来进行伤害的结算和各种buff的效果结算
```

我能够进一步在这个展示的卡片上悬停在 【火球术】 上，它能够显示
```markdown
火系法术，造成火焰伤害，并有几率附加灼烧效果
```

这样就能够非常清晰地展示出每个词条的定义和相关信息，而不需要重复定义。


## 问题回复：
Q1. “的定义” 后缀是否总是可选？是的话我用单一 pattern 加 (的定义)? 处理即可。
A1. 问出这个问题代表你没有理解模板的功能，这里的 “的定义只” 是一个示例，并不具有特别意义。

Q2. 占位符命名是否只会用简单标识（A / B / 英文单词）？还是可能出现中文（例如 {技能}）？（影响提取与 pattern 可靠性）
A2. 占位符命名可以使用简单标识，也可以使用中文，具体取决于模板的设计和使用场景。

Q3. 是否需要在悬浮卡片里额外显示一块“参数列表”区域？
A3. 可以考虑展示，这个优先级不高，你可以考虑先实现基本功能。

Q4. 是否希望当引用里写的是 【使用{滑移}替换{基础移动方式}】（参数内部没加【】）时，在生成的定义正文里自动把参数值再包成【滑移】用于可点击？还是保持原样只在作者显式写 {【滑移】} 时才可点击？
不用，必须要显式写 {【滑移】} 才能点击。这样能够避免混淆。

Q5. 一个模板 alias 是否可能在不同 scope 下重复出现且需要按 scope 匹配？
A5. 是的，一个模板 alias 只应该在 其所属的 scope 下匹配，或者如果不声明 scope，则在所有 scope 下匹配。模板其实和 定义类似，不过它能够拥有空缺的占位符。

Q6. 你是否需要把“模板本身（未替换）”也作为一个可悬停条目（例如显示占位符形式的原始定义）？
A6. 我没有理解你的问题，模板本身作为一个可悬停条目指的是 当使用 内联定义时，也和 定义词条一样能够悬停展示吗？如果是这样，那么是的，我希望能够悬停展示。
