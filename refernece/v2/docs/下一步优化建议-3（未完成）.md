### 空缺模板匹配

好像这个功能我传达的不是很到位

这是 target.md 中的描述：
> 我希望能够使用一些特殊的语法来定义带有空缺的模板，比如说在文档中提到 “使用 {滑移} 替换 {基础移动方式} 的定义”，那么当鼠标悬停在 {滑移} 上时，能够显示 “滑移” 的定义和相关信息，而当鼠标悬停在 {基础移动方式} 上时，能够显示 “基础移动方式” 的定义和相关信息。当 悬浮到 使用 {滑移} 替换 {基础移动方式} 的定义 时，能够显示 “使用 {} 替换 {} 的定义” 的定义和相关信息。

> 比如
> 使用A替换B.md
> ```md
> ---
> alias: 
>     - 使用{A}替换{B}的定义
> ---
> ## 概述
> 使用 {A} 替换 {B} 指的是，将某个模式 {B} 的逻辑替换为使用 {A}。
> ```
> 
> 之后在其他地方使用该定义
> ```md
> 在游戏中，玩家可以【使用 {滑移} 替换 {基础移动方式} 的定义】。
> ```
> 
> 之后当鼠标悬停在 “使用 {滑移} 替换 {基础移动方式} 的定义” 上时，能够显示下面内容
> ```
> ## 概述
> 使用 滑移 替换 基础移动方式 指的是，将某个模式 基础移动方式 的逻辑替换为使用 滑移。
> ```
> 
> 如果使用定义嵌套：
> ```md
> 在游戏中，玩家可以【使用 {【滑移】} 替换 {【基础移动方式】} 的定义】。
> ```
> 那么当鼠标悬停在 “使用 {【滑移】} 替换 {【基础移动方式】} 的定义” 上时，能够显示下面内容
> ```
> ## 概述
> 使用 【滑移】 替换 【基础移动方式】 指的是，将某个模式 【基础移动方式】 的逻辑替换为使用 【滑移】。
> ```
> 
> 之后在这个内容上进一步悬停在 【滑移】 上时，能够显示 滑移 的定义和相关信息，而悬停在 【基础移动方式】 上时，能够显示 基础移动方式 的定义和相关信息。


这里可能会有点难以理解，假设我们不支持空模板匹配，那么：
```md
在游戏中，玩家可以【使用{滑移}替换{基础移动方式}】。
```

应该找不到对应的定义，因为我们只有一个 【使用{}替换{}】的定义，而没有 【使用{滑移}替换{基础移动方式}】的定义。

但是如果我们支持空模板匹配，那么 不管 {} 里面的内容是什么，它都能匹配到 【使用{}替换{}】的定义。

更进一步的，我们可以获取 {} 里面的内容，并且在鼠标悬停时展示的悬浮卡片中，将 {} 内的东西 替换为实际的内容。

比如
替换的定义.md
```md
---
alias: 
    - 使用{A}替换{B}
---
## 概述
使用 {A} 替换 {B} 指的是，将某个模式 {B} 的逻辑替换为使用 {A}。
```

然后在其他地方使用该定义
```md
在游戏中，玩家可以【使用{滑移}替换{基础移动方式}】。
```

当鼠标悬停在 “使用{滑移}替换{基础移动方式}” 上时，能够显示下面内容
```
## 概述
使用 滑移 替换 基础移动方式 指的是，将某个模式 基础移动方式 的逻辑替换为使用 滑移。
```

更进一步的，如果 {} 里面的 不是简单的字符串，而是一个词条引用，比如
```md
在游戏中，玩家可以【使用{【滑移】}替换{【基础移动方式】}】。
```
那么当鼠标悬停在 “使用{【滑移】}替换{【基础移动方式】}” 上时，能够显示下面内容
```
## 概述
使用 【滑移】 替换 【基础移动方式】 指的是，将某个模式 【基础移动方式】 的逻辑替换为使用 【滑移】。
```

这里面的 【滑移】 和 【基础移动方式】 将会被展示为嵌套定义的词条，当鼠标悬停在 【滑移】 上时，能够显示 滑移 的定义和相关信息，而悬停在 【基础移动方式】 上时，能够显示 基础移动方式 的定义和相关信息。

我的质朴的设计思路就是，当遇到这种的时候，解析前对于源文本 进行一个正则替换，将 {A} 替换为 外部定义的 A 的内容，将 {B} 替换为 外部定义的 B 的内容。之后在像正常处理词条一样处理这个文本。这个想法不一定对，请你谨慎听取。
